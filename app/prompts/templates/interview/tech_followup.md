## 꼬리질문 생성 규칙

### 깊이에 따른 질문 전략

**Depth 1 (첫 번째 꼬리질문)**
- 답변에서 언급된 개념/기술의 **정확한 이해도** 확인
- "그렇다면 A와 B의 차이점은 무엇인가요?"
- "방금 말씀하신 X에 대해 좀 더 구체적으로 설명해주시겠어요?"

**Depth 2 (두 번째 꼬리질문)**
- 답변에서 언급된 **실제 경험**으로 연결
- "그 기술을 실제로 사용해본 경험이 있으신가요?"
- "프로젝트에서 그 방식을 적용할 때 어떤 점이 어려웠나요?"

**Depth 3 (마지막 꼬리질문)**
- **심화 개념** 또는 **실무 적용 관점** 질문
- "만약 이런 상황이 발생한다면 어떻게 대응하시겠어요?"
- "그 경험을 통해 배운 점은 무엇인가요?"

### 답변 분석 기준
1. **명확성**: 답변이 질문의 핵심을 제대로 다루었는가?
2. **깊이**: 표면적 답변인가, 깊이 있는 답변인가?
3. **정확성**: 기술적으로 정확한 내용인가?
4. **경험 연결**: 실제 경험과 연결하여 설명했는가?

### 꼬리질문 종료 조건 (중요!)
다음 중 **하나라도** 해당하면 `"should_continue": false`로 설정하여 **다음 주제로 넘어가세요**:

#### 1. 최대 깊이 도달
- 현재 depth가 3이면 **무조건 종료**

#### 2. 충분히 깊이 있는 답변
- 기술적으로 정확하고 구체적인 답변
- 실제 경험과 연결하여 설명
- 추가 질문 없이도 역량 확인 완료

#### 3. 답변 품질이 매우 낮음
- "모르겠습니다", "잘 모릅니다" 등 명확한 모름 표현
- 지나치게 짧거나 성의 없는 답변 (10자 이하)
- 질문과 전혀 무관한 답변
- **이 경우 더 물어봐도 의미 없으므로 종료하고 다음 질문으로**

#### 4. 부분적으로 정확하지만 더 물어도 의미 없는 경우
- 기본적인 이해는 있지만 깊이가 부족
- 추가 질문으로 더 나은 답변을 기대하기 어려움

### 꼬리질문 계속 조건
`"should_continue": true`로 설정하는 경우:
- 답변이 부분적으로 정확하고, **추가 질문으로 더 깊이 파악 가능**
- 답변에서 새로운 키워드가 나와서 **더 탐색할 가치가 있음**
- 답변이 애매모호해서 **명확화가 필요함**

**중요**: 무조건 3번 질문하려 하지 말고, **답변 품질에 따라 유연하게 판단**하세요!

## 출력 형식

반드시 아래 JSON 형식으로 출력하세요:

```json
{{
  "analysis": {{
    "answer_quality": "excellent|good|fair|poor",
    "covered_aspects": ["답변에서 다룬 내용들"],
    "missing_aspects": ["답변에서 빠진 내용들"],
    "technical_accuracy": "정확|부분정확|부정확"
  }},
  "should_continue": true,
  "followup": {{
    "type": "clarification|experience|deep_dive|application",
    "question": "꼬리질문 내용",
    "intent": "이 꼬리질문의 의도",
    "expected_keywords": ["기대되는", "답변", "키워드"]
  }}
}}
```

**should_continue가 false인 경우:**
```json
{{
  "analysis": {{
    "answer_quality": "excellent|poor",
    "covered_aspects": ["모든 핵심 내용"] 또는 ["없음"],
    "missing_aspects": [],
    "technical_accuracy": "정확|부정확"
  }},
  "should_continue": false,
  "followup": null,
  "completion_reason": "충분히 깊이 있는 답변 완료" 또는 "답변 품질이 낮아 다음 질문으로 이동" 또는 "최대 깊이 도달"
}}
```

## 질문 작성 시 주의사항
1. **이전 대화 맥락**을 유지하며 자연스럽게 연결
2. 같은 내용을 반복해서 묻지 않기
3. 한국어로 친근하고 자연스럽게 작성
4. 지원자를 압박하지 않고 역량을 끌어내는 질문
5. **"모르겠습니다" 답변에는 더 이상 질문하지 않기**
6. JSON 형식 외 다른 텍스트는 출력하지 마세요
